## A. 検索機能の拡張

### A1. 必須文字指定（除外の逆）｜規模: 小〜中
- **概要**: 「カを含む語だけ」のように、必ず含むべき文字を指定して絞り込む。
- **価値**: 盤面の余り文字を消化したい場面で有用。`＊カ＊` パターンでは複数文字・位置不問の指定ができない。
- **実装メモ**: `normalized LIKE '%カ%'` の AND 連結。`exclude` パラメータ（[backend/api.php](../backend/api.php)）とまったく同じ型で実装できる。UI も除外文字の折りたたみパネルに同居可能。

### A2. 交差ヘルパー（？位置の文字分布）｜規模: 大
- **概要**: パターンの各 `？` 位置に「どの文字が何件入りうるか」の一覧を表示する。
- **価値**: 交差マスの文字選びを直接支援する本命機能。候補を1つずつ試す手間が消える。
- **実装メモ**: API に `mode=charstats` を追加し `SELECT substr(normalized, N, 1), COUNT(*) ... GROUP BY 1` で集計。`*` なしパターンなら len 固定で高速。UI 設計（表示場所・モバイルでの見せ方）が重いので要ブレスト。

### A3. 独自辞書の充実ワークフロー｜規模: 小
- **概要**: words_addition.txt への単語追加を、重複チェック・フォーマット検証付きで行う CLI スクリプト。
- **価値**: 辞書の質がアプリの価値そのもの。追加作業の心理的コストを下げる。
- **実装メモ**: scripts/ に追加。既存の `/import-words-db` スキル・[backend/import.php](../backend/import.php) と連携。

## B. UX・パフォーマンス

### B1. コピー形式のカスタマイズ｜規模: 小
- **概要**: 拗音・促音展開の ON/OFF、見出し行（【パターン】）の有無、区切り文字（改行/読点）を設定可能に。
- **価値**: コピー先（メモ・スプレッドシート・作問ツール）によって欲しい形式が違う。
- **実装メモ**: localStorage 保存。[frontend/app.js](../frontend/app.js) の `copyAllResults()` と読みボタンコピーに分岐追加。設定 UI の置き場所（ヘルプ的なポップオーバー？）は要検討。

### B2. PWA インストール促進｜規模: 小
- **概要**: `beforeinstallprompt` を捕捉し、控えめなインストール導線を出す（ヘッダー or 初回ガイド内）。
- **価値**: リピーターのホーム画面定着。PWA 対応済みなのに導線がない。

### B3. 外部辞書リンクの拡充｜規模: 小
- **概要**: 表記リンクの遷移先を Google 検索に加えてコトバンク等から選べるように。
- **価値**: カギ（ヒント文）作成時の意味確認が早くなる。
- **実装メモ**: `buildWordItem()` のリンク生成を設定化。

### B4. Noto Sans JP の有効化検討｜規模: 小
- **概要**: 同梱済みの Noto Sans JP を有効化し、システムフォントと読み比べて採否判断。
- **実装メモ**: [frontend/style.css](../frontend/style.css) 冒頭コメントに切替手順あり。フォント120ファイルの転送量と見た目のトレードオフ。

### B5. 完全オフライン検索｜規模: 大・要方針判断
- **概要**: words.db を sql.js（WASM）＋ IndexedDB でクライアントに持ち、API 不通でもフル検索できるようにする。
- **価値**: 電波のない場所でも使える・検索レイテンシゼロ。
- **懸念**: **依存ゼロ方針（CLAUDE.md）と衝突**するため要相談。DB（10MB 級）の初回ダウンロードも課題。着手前にブレスト必須。

### B6. 大量結果の描画最適化｜規模: 中・優先度低
- **概要**: 5,000件 DOM の仮想スクロール化。
- **メモ**: 既に `content-visibility` 適用済みで現状問題は出ていない。**実測で問題が確認されてから**着手する。

## C. SEO・集客

### C1. JSON-LD 構造化データ｜規模: 小
- **概要**: index.html に `WebApplication`、help.html に `FAQPage` スキーマを追加。
- **価値**: 検索結果での見え方改善。現状 OGP は完備だが構造化データはゼロ。

### C2. findword サブドメインの計測整備｜規模: 小（設定作業）
- **概要**: Search Console へ findword.crossword-builder.com のプロパティを追加。アプリの GA4 プロパティを MCP に接続。
- **価値**: **2026-06 の改善検討時、接続済みプロパティに本体サイトのデータしかなくデータドリブン提案ができなかった**。以後の改善判断の土台になる。

### C3. help.html の SEO 活用｜規模: 小
- **概要**: 現在 `noindex,follow`。内容を拡充して index 許可し、「クロスワード 単語検索 使い方」系クエリの受け皿にする。

### C4. OGP 画像の更新｜規模: 小
- **概要**: og-image.jpg を新機能（除外文字・シャッフル・パターン解釈表示）反映後のスクリーンショットに差し替え。

## D. 運用・品質の自動化

### D1. バージョン更新の自動化｜✅ 実装済み（2026-06-10）
- **概要**: `?v=YYYYMMDDXX`（index/help/privacy の3ファイル）と sw.js の `VERSION` を一括更新するスクリプト。
- **実装**: [scripts/bump-version.php](../scripts/bump-version.php)。自動採番 / `--dry-run` / `--set`（巻き戻し）対応。

### D2. E2E スモークテストの資産化｜規模: 中
- **概要**: Playwright で手動検証した項目をテストスクリプト化。検索・除外文字・シャッフルのページング重複なし・まとめてコピー・**フォーム暗黙送信**（実 Enter キーで submit が発火するか）。
- **価値**: iOS 完了キーでキーボードが閉じなくなった回帰（2026-06-10 発生）のような問題を機械検知できる。デプロイ前に1コマンドで実行。

### D3. API 後方互換スナップショットテスト｜✅ 実装済み（2026-06-10）
- **概要**: 代表17ケース（完全一致 / ？のみ / ＊あり / 数字+＊ / exclude / shuffle / ページング / エラー系）の API 応答を保存し、api.php 変更時に diff 比較。
- **実装**: [scripts/api-snapshot.php](../scripts/api-snapshot.php)（check/update）。スナップショットは scripts/api_snapshots/ に保存。パラメータは stdin 経由で渡し Windows の文字化けを回避（scripts/api-query-runner.php）。

### D4. クライアントエラーロギング｜規模: 小
- **概要**: `window.onerror` / `unhandledrejection` → GA4 カスタムイベント送信（本番ドメインのみ）。
- **価値**: 実ユーザー環境（特に iOS Safari）のエラーを可視化。外部サービス不要で依存ゼロ維持。

### D5. デプロイ差分リストの自動生成｜✅ 実装済み（2026-06-10）
- **概要**: デプロイごとに git タグ（`deploy-YYYYMMDD-HHMM`）を打ち、前回タグとの diff から FTP アップロード対象リストを出力。
- **実装**: [scripts/deploy-diff.php](../scripts/deploy-diff.php)。frontend/・backend/*.php を対象判定、words.txt 変更時は DB 再構築を注意喚起。アップロード後に `--mark` でタグ付け。

### D6. GitHub Actions CI｜規模: 中
- **概要**: push 時に `php -l` ＋ D2/D3 のテストを実行。
- **メモ**: CLAUDE.md に「CI は未構成」と明記されている項目。D2/D3 が先。
